See you on the flip side

For a recent project, I needed a way of making an ordinary HTML element appear to flip over and reveal something on the other side. Fortunately, the project was cutting edge browser only — Chrome, Safari, Firefox and IE9+ were listed as requirements, and the technique below also works nicely on Android and iOS devices.

Naturally, in the absence of any idea of how to solve the problem I Googled it. Finding only slightly disappointing (but actually very good if you need something that works in older browsers) or very nice, 3D but not IE9 friendly efforts, I set about creating my own.

I quickly discovered CSS3’s transform property. As with many exciting bits of CSS, the main browser vendors have yet to decide on a standard, so they’ve all provided their own -vendor-transform equivalents. Happily, they all seem to be exactly the same, at least as far as I was using them.

Many transformations are available, but the two I was interested in are scale and rotate. For example…

#myelement { -webkit-transform : scale(0.6, 1) rotate(4deg); -moz-transform : scale(0.6, 1) rotate(4deg); -ms-transform : scale(0.6, 1) rotate(4deg); }

…has the nice effect of making an element look a bit like it is in the process of being flipped over. Have a look here, if you’re on one of the above browsers.

The next step was to animate those properties so that my element looked like it was being flipped. In fact, that process needs two elements — #front and #back, to represent each side of an imaginary piece of card that’s being flipped. The #front starts life visible, while #back is hidden and transformed into a rotated, squashed version of itself.

To flip the ‘card’, we just need to reduce the x-axis scale of #front, while rotating it. Once the scale is reduced to a slither, we do the opposite transformation on #back. You can see this in action using jQuery’s animate() function here, or embedded below –
